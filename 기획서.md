# 실시간 통역 웹사이트 — 기획 및 구현 계획

## Context (배경)

해외 출장 회의에서 사용할 **웹 기반 실시간 통역 도구**를 신규 제작한다. 작업 디렉토리(`C:\Users\sales\workspace\Realtime_Translator`)는 현재 비어 있는 그린필드 프로젝트다.

OpenAI의 신규 모델 **`gpt-realtime-translate`**(2026-05-07 출시)를 사용한다. 이 모델은 음성을 받아 실시간으로 번역 음성 + 번역 텍스트(transcript delta)를 스트리밍한다. 회의 중 음성 출력은 방해가 되므로, **번역 음성은 기본 OFF로 두고 대형 텍스트 창**으로 번역 결과를 보여주는 것이 핵심 설계 목표다.

### ⚠️ 중요 — ChatGPT Pro 플랜과 API 과금은 별개
사용자는 ChatGPT Pro 20x 플랜을 사용 중이나, **이 구독은 OpenAI API 사용료를 포함하지 않는다.** API는 `platform.openai.com`에서 별도의 결제수단 등록과 크레딧 충전이 필요하며, Pro 구독 토큰과 무관하게 별도 청구된다. 또한 `gpt-realtime-translate`는 **무료 등급(Free tier)에서 사용 불가**하며, 최소 Tier 1(API 계정에 결제 후 도달) 이상이 필요하다.
- 모델 요금: **$0.034 / 분** (오디오 스트리밍 시간 기준, 침묵 구간 포함)
- 동시 양방향 모드는 세션 2개 → 약 **$0.068 / 분**

### 확정된 설계 결정
- **보안 구조**: Google Apps Script 백엔드 (시트에 연결된 Apps Script를 웹 앱으로 배포) + 다층 방어
- **양방향**: 단방향 / 동시 양방향 / 방향전환 토글 — 3개 모드, 회의 중 모드 변경 버튼으로 전환 가능
  - **대면 회의 기본값 = 토글 양방향** (마이크 1개로 양쪽 발화가 섞이므로)
  - **동시 양방향은 오디오 분리가 가능한 상황(화상 회의 등)에서 권장**
- **음성 입력**: 마이크(대면 회의) + 탭/시스템 오디오(화상 회의) 모두 지원

## 1. 사용 API 스펙 요약 (`gpt-realtime-translate`)

| 항목 | 내용 |
|---|---|
| 기능 | 스트리밍 음성→음성 통역 (전문 통역사 음성으로 학습) |
| 입력 | 오디오만 (24kHz PCM16) — 텍스트/이미지 입력 불가 |
| 출력 | 번역 오디오 + 텍스트 transcript delta |
| 입력 언어 | 70개+ 자동 감지 |
| 출력 언어 | 13개 — 한국어·영어·중국어·일본어·스페인어·프랑스어·독일어·포르투갈어·러시아어·힌디어·인도네시아어·베트남어·이탈리아어 |
| 전용 엔드포인트 | `/v1/realtime/translations` (일반 `/v1/realtime`과 다름) |
| 미지원 | 커스텀 프롬프트·용어집(glossary)·function calling — 통역 전용 |

### 연결 흐름 (WebRTC, 브라우저용)
1. 서버(Apps Script)가 `POST /v1/realtime/translations/client_secrets` 호출 → 단기 client secret 발급
2. 브라우저가 오디오 트랙 + 데이터채널(`oai-events`) 포함 `RTCPeerConnection` 생성
3. SDP offer를 `POST /v1/realtime/translations/calls` (Authorization: Bearer `<client_secret>`)
4. 데이터채널 이벤트 수신:
   - `session.output_transcript.delta` → 번역된 텍스트
   - `session.input_transcript.delta` → 원문 텍스트
   - `session.output_audio.delta` → 번역 음성 (기본 미재생)
5. 세션 설정: `session.audio.output.language` = 목표 언어, `session.audio.input.transcription.model` = `gpt-realtime-whisper`

> **음성 스트리밍(WebRTC)은 브라우저↔OpenAI 직결**이며 Apps Script를 거치지 않는다. Apps Script는 세션당 1회 토큰 발급에만 사용되어 통역 지연에 영향을 주지 않는다.

```
│             │  ③단기 client secret            │ - OPENAI_API_KEY     │

│             │                                  └──────────┬───────────┘
│             │  ④WebRTC 음성 스트리밍 (직결)               │ ②
│             │ ───────────────────────────────────────────▼──────────┐
└─────────────┘                                  │  OpenAI API         │
                                                  │  realtime/translations
                                                  └─────────────────────┘
```

### 보안 설계 (Apps Script 방식) — 다층 방어

토큰 발급 엔드포인트는 공개 URL이므로, 앱 비밀번호 하나에 의존하지 않고 **4중 방어**를 적용한다.

1. **OPENAI_API_KEY**: Apps Script의 Script Properties에 보관 → 브라우저에 절대 전달되지 않음. 시트 셀에는 저장하지 않는다.
2. **APP_PASSWORD (공유 비밀키)**: 사용자가 정한 임의 문자열. 프론트엔드가 모든 토큰 요청에 동봉하고, `doPost`가 일치 여부를 검증해 통과한 요청에만 토큰을 발급. 유출돼도 OpenAI 키는 안전하며 Apps Script에서 값만 교체하면 된다.
3. **일일 발급 제한**: `doPost`가 Script Properties에 날짜별 발급 횟수를 누적하고, 상한(예: 100회/일) 초과 시 차단 → 비밀번호가 유출돼도 대량 악용 불가.
4. **OpenAI 예산 한도**: `platform.openai.com` 대시보드에서 월 사용 한도(hard limit)와 알림 임계치를 설정 → 모든 방어가 뚫려도 최대 손실 상한 고정.
5. **로그 기록**: `doPost`가 발급 시각·요청 IP·목표 언어를 Apps Script 로그 또는 시트의 별도 로그 탭에 기록 → 이상 사용 사후 추적.

- 프론트엔드 설정값(Apps Script URL, 앱 비밀번호)은 브라우저 `localStorage`에 저장(최초 1회 입력).
- CORS: 프론트엔드는 `Content-Type: text/plain`으로 POST(프리플라이트 회피), Apps Script는 `ContentService`로 JSON 응답.

## 3. 통역 모드 설계

상단 **모드 선택 버튼**으로 3개 모드를 회의 중에도 즉시 전환한다.

| 모드 | 동작 | 세션 수 | 비용 | 권장 상황 |
|---|---|---|---|---|
| **단방향** | 상대 언어 → 한국어 (대형 1패널) | 1 | $0.034/분 | 듣기 위주, 발표 청취 |
| **토글 양방향** | 한 번에 한 방향, 스페이스바/버튼으로 전환 (1패널) | 1 | $0.034/분 | **대면 회의 기본값** — 마이크 1개로 양쪽 발화가 섞이는 환경 |
| **동시 양방향** | 상대→한국어, 한국어→상대 동시 (2패널) | 2 | ~$0.068/분 | **오디오 분리 가능 시** — 화상 회의(탭 오디오+마이크 분리), 별도 마이크 2개 등 |

- **동작 원리**: 모델은 목표 언어와 다른 모든 음성을 목표 언어로 번역하고, 이미 목표 언어인 음성은 번역하지 않는다.
- **기본값**: 대면 회의에서는 **토글 양방향**으로 시작한다. 단일 마이크로 동시 양방향을 쓰면 두 세션이 같은 발화를 받아 오인식·중복·2배 비용이 발생할 수 있기 때문이다.
- **동시 양방향 권장 조건**: 오디오 채널을 방향별로 분리할 수 있을 때만 권장한다.
  - 화상 회의: 세션 A는 탭 오디오(상대 음성), 세션 B는 마이크(내 음성)
  - 대면 회의: 발화자별 마이크 2개를 각 세션에 배정할 수 있을 때
- **모드 전환 처리**: 전환 시 불필요한 세션은 정리(close), 필요한 세션은 새로 생성. 활성 세션은 유지해 끊김 최소화.
- 대면 회의에서 동시 양방향을 선택하면 위 한계를 알리는 안내 문구를 표시한다.

## 5. 편의 기능 (제안 포함)

1. **실시간 비용 미터 + 세션 타이머** — 활성 세션 수 × 경과 시간 × $0.034로 누적 비용 표시 (과금 통제)
2. **일시정지/재개 (스페이스바)** — 침묵 구간도 과금되므로 휴식 시 세션 정지로 비용 절감
3. **회의록 저장/내보내기** — 원문+번역+타임스탬프를 `.txt`/`.md`로 저장. 저장 직전 *"통역 결과에 오류가 있을 수 있으며, 회의 내용에 민감·기밀 정보가 포함될 수 있으니 취급에 주의하세요"* 안내 문구를 표시. 회의록은 사용자가 명시적으로 저장할 때만 파일로 기록(기본은 메모리에만 유지)
4. **상대방 패널 미러 모드** — 노트북을 가운데 두고 마주 앉을 때 패널을 180° 회전
5. **보조 창 팝업** (`popup.html`) — 상대방용 패널을 별도 창으로 띄워 보조 모니터/태블릿에 표시
6. **자동 재연결** — 네트워크 끊김 감지 시 세션 재수립
7. **글꼴 크기 조절 / 다크·고대비 모드 / 전체화면**
8. **번역 음성 기본 OFF** — 회의 방해 방지. ON으로 켤 경우 **출력 장치 선택을 필수**로 요구(이어폰 등)하여 회의실 스피커로 새어 나가지 않도록 강제
9. **원문 언어 자동감지 배지** 표시
10. **키보드 단축키** — Space(정지), F(전체화면), M(모드 전환) 등

## 7. 구현 단계 (빌드 순서)

위 파일 구조를 다음 순서로 구현한다 (파일별 책임은 6장 참조, 여기서는 핵심 작업만 명시).

1. **`Code.gs`** — `doPost`: 앱 비밀번호 검증 → 일일 발급 한도 확인 → `client_secrets` 호출 → 로그 기록 → client secret 반환. 입력 transcription(`gpt-realtime-whisper`)·`noise_reduction: near_field` 포함. `doGet`은 헬스체크.
2. **`config.js` + 설정 패널** — Apps Script URL·앱 비밀번호 입력/저장.
3. **`audio.js`** — `getUserMedia`/`getDisplayMedia` 캡처, 장치 열거·선택.
4. **`session.js`** — 토큰 요청 → WebRTC offer/answer → 데이터채널 이벤트 파싱 → 콜백.
5. **`ui.js`** — 패널·확정/delta 구분·자동 스크롤·미러·테마.
6. **`app.js`** — 3개 모드 생성/전환, 세션 수명주기, 단축키.
7. **`record.js`** — 비용 미터·타이머, 회의록 누적·내보내기(저장 전 안내 문구).
8. **`popup.html`** — `BroadcastChannel`로 메인 창과 동기화.
9. **`style.css`** — 회의실 가독성 중심 디자인, 다크/고대비.

## 8. 제약사항 및 주의점

- **API 결제 필수**: `platform.openai.com`에 결제수단 등록 + 크레딧 충전, Tier 1 이상 필요 (Free tier 사용 불가).
- **용어집/커스텀 프롬프트 미지원**: 회사·제품 고유명사를 강제 지정 불가 (모델 한계).
- **목표 언어와 같은 언어는 번역 안 됨**: 혼용 발화(예: 한국어 속 영어 단어) 일부 누락 가능.
- **HTTPS/localhost 필수**: `getUserMedia`/`getDisplayMedia`는 보안 컨텍스트에서만 동작. `file://` 직접 열기는 불가 → 로컬 서버로 실행해야 함.
- **출력 언어 13개 한정**: 목록 외 언어는 지원 불가.
- 침묵 구간도 과금되므로 회의 휴식 시 일시정지 권장.

## 9. 검증 방법

1. **Apps Script 배포**
   - 새 Google 스프레드시트 → 확장프로그램 → Apps Script → `Code.gs` 붙여넣기
   - Script Properties에 `OPENAI_API_KEY`, `APP_PASSWORD` 등록
   - "웹 앱"으로 배포(실행: 본인 / 액세스: 모든 사용자) → URL 복사
2. **로컬 실행**: 프로젝트 폴더에서 `python -m http.server 8000` → `http://localhost:8000` 접속
3. **설정**: ⚙ 설정에 Apps Script URL·앱 비밀번호 입력
4. **단방향 테스트**: 영어로 말하기 → 대형 창에 한국어 번역 텍스트 실시간 표시 확인
5. **토글 테스트**: 스페이스바/버튼으로 방향 전환, 영어·한국어 발화가 올바른 방향으로 번역되는지 확인
6. **동시 양방향 테스트**: 오디오 분리 환경(탭 오디오+마이크)에서 두 패널에 각각 표시 확인
7. **모드 전환**: 회의 중 모드 버튼으로 토글 ↔ 동시 양방향 전환 시 끊김·오류 없는지 확인
8. **편의 기능**: 비용 미터 증가, 일시정지 시 미터 정지, 회의록 저장 전 안내 문구, 미러/팝업 창, 자동 재연결 확인
9. **보안 확인**:
   - 브라우저 개발자도구 네트워크 탭에서 OpenAI API 키가 노출되지 않고 단기 client secret만 보이는지 확인
   - 잘못된 앱 비밀번호로 토큰 요청 시 거부되는지 확인
   - 일일 발급 한도 초과 시 차단되는지 확인

### 실제 회의 리허설 (배포 전 필수)

기능 테스트와 별개로, 실사용 조건을 모사한 리허설을 수행한다.

- **대면 회의 시나리오**: 두 사람이 노트북 마이크 하나로 영어↔한국어 대화 → 토글 양방향으로 발화자가 바뀔 때마다 방향 전환이 자연스러운지, 지연이 회의 흐름을 해치지 않는지 확인
- **화상 회의 시나리오**: Zoom/Teams 통화에 참여해 탭 오디오 캡처 → 동시 양방향으로 양쪽 패널이 동시에 갱신되는지 확인
- **회의실 가독성**: 1~2m 거리에서 대형 번역 텍스트가 읽히는지, 다크/고대비 모드·폰트 크기 확인
- **장시간 안정성**: 20~30분 연속 사용 시 끊김·재연결·메모리 확인, 비용 미터 누적값이 실제 청구와 근사한지 확인
- **소음 환경**: 회의실 잡음·동시 발화 상황에서 오인식 정도 확인
- **상대방 패널 운용**: 미러 모드 또는 팝업 창을 보조 기기에 띄워 상대방이 실제로 읽을 수 있는지 확인
